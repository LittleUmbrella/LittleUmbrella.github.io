Deployment

Steps

  1. Get the bits
  2. Build (Visual Studio 2010)
  3. Install
  4. configure

Bits

Repository: \\TMFRAMEWORK (TFS)

Get latest of $/1CF Program/Workflow/CloseAccount

Build 

$/1CF Program/Workflow/CloseAccount/Trunk/src/BECU.CloseAccount.sln

Installation

  1. Create a local folder on web server(s) to hold the web site files
  2. Copy the build output of BECU.CloseAccount.Web.UI, i.e. copy it's bin folder to the server (the publish shortcut in Visual Studio will do fine).
  3. Copy web.config, and all .svc files if not copied from above.

Configuration

IIS Configuration

  • App Pool I'll leave this up to you, but
    • needs no special access/rights.
    • .net 4.0
  • Web Site Standard .NET Web Site configuration.
    • Read and script rights and its own app pool.

Web.Config Configuration

The site communcates to the BECU.CloseAccount.WebAPI and BECU.CloseAccount.ServiceHost projects, which should be siblings to this.  That communication is not ready at the moment, so no changes are necessary here.
